# Менеджер академических дедлайнов (Study Dependency Tracker)

## Идея проекта
Сделать приложение для планирования учебы, которое учитывает зависимости между задачами. Суть в том, что обычные списки дел не показывают, что, например, «нельзя начать делать дз номер 3, пока не выучены темы». Приложение будет строить граф задач и находить главный путь — задания, которые нужно делать прямо сейчас, чтобы не завалить финальный дедлайн и не осаться без баллов.

## Инструкция по запуску
Для запуска приложения необходимо создать виртуальное окружение и установить зависимости:

1. Создайте виртуальное окружение:
   `python3 -m venv venv`
2. Активируйте окружение:
   - Mac/Linux: `source venv/bin/activate`
   - Windows: `venv\Scripts\activate`
3. Установите зависимости:
   `pip install -r requirements.txt`
4. Запустите приложение из корневой папки проекта:
   `python main.py`

## Используемые библиотеки (не из программы курса)
- NetworkX — для работы с графами и алгоритмами поиска связей.
- PyQt6 — для создания надежного и нативного десктопного интерфейса.
- Matplotlib — для визуализации графа внутри окна приложения.

## Итерация 1
1. Создал основу приложения: объекты задач (название, сколько часов займет, дедлайн) и возможность связывать их друг с другом.
2. Делал базовое десктопное окно на PyQt6 с формами добавления задач и связей, а также обработкой ошибок ввода.
3. Реализовал проверку на циклы (чтобы нельзя было поставить задачу А зависимой от Б, если Б уже зависит от А).
4. Настроил базовую статичную визуализацию графа, чтобы сразу видеть дерево связей и подсветку главного пути.

## Итерация 2
1. Добавил расчет самого длинного пути (главного пути): программа подсвечивает задачи, которые нельзя откладывать.
2. Создал возможность перетаскивать узлы мышкой в реальном времени для удобной компоновки на экране.
3. Добавил сохранение и загрузку учебного плана в JSON: задачи, связи между ними и базовые параметры графа, которые будут сохраняться между запусками приложения.
4. Покрыл логику GraphManager юнит-тестами (add_task, цикл, longest path, JSON).
5. Написал скрипт профилирования для проверки производительности алгоритмов на графах размером до 2000 узлов.
