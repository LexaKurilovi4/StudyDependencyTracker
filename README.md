# Менеджер академических дедлайнов (Study Dependency Tracker)

## Идея проекта
Сделать приложение для планирования учебы, которое учитывает зависимости между задачами. Суть в том, что обычные списки дел не показывают, что, например, «нельзя начать делать дз номер 3, пока не выучены темы». Приложение будет строить граф задач и находить главный путь — задания, которые нужно делать прямо сейчас, чтобы не завалить финальный дедлайн и не осаться без баллов.

## Используемые библиотеки (не из программы курса)
- NetworkX — для работы с графами и алгоритмами поиска связей.
- CustomTkinter — для создания интерфейса программы(кнопки, тема и т.д.).

## Итерация 1
1. Создам основу приложения: объекты задач (название, сколько часов займет, дедлайн) и возможность связывать их друг с другом.
2. Сделаю базовое десктопное окно с формой добавления задач и списком дел.
3. Реализую проверку на циклы (чтобы нельзя было поставить задачу А зависимой от Б, если Б уже зависит от А).

## Итерация 2
1. Добавлю расчет самого длинного пути (главного пути): программа будет подсвечивать задачи, которые нельзя откладывать.
2. Сделаю визуализацию графа — прямо в приложении будет видно дерево связей между предметами и темами.
3. Реализую сохранение и загрузку всех задач через JSON, чтобы план не терялся при перезагрузке.
